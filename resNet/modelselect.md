| 모델          | 파라미터 | 블록 구조      | FC 입력 차원 | 판단                                  |

## 3. ResNet-50 추천 근거

### 이유 1 — 블록 구조: Bottleneck vs BasicBlock

| 구분             | ResNet-34 (BasicBlock) | ResNet-50 (Bottleneck) |
| ---------------- | ---------------------- | ---------------------- |
| 블록 내부        | `3×3 Conv → 3×3 Conv`  | `1×1 → 3×3 → 1×1 Conv` |
| 비선형 변환 깊이 | 얕음                   | 깊음                   |
| 특징 추출 능력   | 보통                   | 높음                   |

전입신고서처럼 **필드 위치와 텍스트 패턴이 중요한 문서 이미지**에서는,
Bottleneck 구조가 더 풍부한 계층적 특징을 추출한다.

### 이유 2 — FC 입력 차원: 2048 vs 512

- ResNet-34 마지막 특징 벡터: **512차원**
- ResNet-50 마지막 특징 벡터: **2048차원** (4배)
- 14개 이진 레이블을 **동시에** 판별하려면 각 필드에 해당하는 특징이 벡터 공간에서 독립적으로 분리되어야 함
- 512차원으로 14개 레이블을 분리하는 것보다 2048차원이 **표현 공간 여유** 측면에서 유리

### 이유 3 — 데이터 규모 적합성

```
ResNet-18  (11M) : 표현력 부족 → 과소적합 위험
ResNet-50  (25M) : 2,799장에 최적 균형 ✅
ResNet-101 (44M) : Fine-tuning 파라미터 과다 → 과적합 위험
```

Fine-tuning 전략(`layer4` + `fc` unfreeze)을 적용하면 실제 학습 파라미터가
전체의 일부로 줄어들어 과적합 위험을 최소화할 수 있다.

## 4. ResNet-50 모델 구성 (참고)

```python
model = models.resnet50(weights=models.ResNet50_Weights.DEFAULT)

# 전체 파라미터 freeze
for param in model.parameters():
    param.requires_grad = False

# layer4 (마지막 Bottleneck 그룹) unfreeze — Fine-Tuning 대상
for param in model.layer4.parameters():
    param.requires_grad = True

# FC 헤드 교체: 2048 → 14
model.fc = nn.Linear(2048, NUM_CLASSES)
```

### 타 모델과 헤드 교체 비교

| 모델           | unfreeze 대상                 | 헤드 교체       | 입력 차원 |
| -------------- | ----------------------------- | --------------- | --------- |
| EfficientNetB4 | `features[-1]`                | `classifier[1]` | 1792      |
| ConvNeXt-Small | `features[6]` + `features[7]` | `classifier[2]` | 768       |
| **ResNet-50**  | **`layer4`**                  | **`fc`**        | **2048**  |

## 5. 결론

> **ResNet-50** 을 선택한다.
> ResNet-34는 BasicBlock + 512차원 특징 벡터로 14개 Multi-label 태스크에서 표현력이 부족하다.
> ResNet-50은 Bottleneck 구조와 2048차원 특징 벡터로 동일한 사전학습 모델군 내에서
> 데이터 규모(2,799장)와 태스크 복잡도(14-label) 양면에서 가장 균형 잡힌 선택이다.
